## 一、选择题答案与解析

---

**1. 答案：B**

**解析：**
- A错误：增大光圈（减小f值）会**减小**景深，使背景更加模糊，而非增大景深。
- **B正确**：景深的定义就是在焦点前后能够保持"足够清晰"的范围。在这个范围内的物体在成像平面上看起来是清晰的。
- C错误：焦距越短，景深越**大**（广角镜头景深深）；焦距越长，景深越浅（长焦镜头容易背景虚化）。
- D错误：景深与多个因素有关：光圈大小、焦距、拍摄距离、传感器尺寸等，不只与光圈有关。

---

**2. 答案：C**

**解析：**
- A错误：杆体细胞约1.2亿个，不是锥体细胞。锥体细胞约600万个。
- B错误：杆体细胞主要分布在视网膜**周边**区域，中央凹处几乎只有锥体细胞。
- **C正确**：杆体细胞对光的敏感度很高，负责暗环境下的视觉（夜视），但不感知颜色差异。这是选择题高频考点。
- D错误：锥体细胞分为三种类型（L型对红光敏感、M型对绿光敏感、S型对蓝光敏感），能区分不同波长的光。

---

**3. 答案：C**

**解析：**
- A错误：BRDF描述的是物体表面的反射特性，与材质密切相关，不是光源的发射特性。
- B错误：BRDF同时依赖于入射方向ωi和出射方向ωr，这是"双向"的含义。
- **C正确**：BRDF（双向反射分布函数）的定义为 fr(ωi, ωr) = dLr(ωr) / (Li(ωi)cosθi dωi)，同时依赖入射方向、出射方向和表面材质。
- D错误：BRDF的单位是sr⁻¹（每球面度），不是瓦特。

---

**4. 答案：C**

**解析：**
- A正确：RGB是线性颜色空间，采用加色模型，用于显示器。
- B正确：CMYK是减色模型（Cyan、Magenta、Yellow、Key/Black），用于印刷。
- **C不正确**：HSV颜色空间是**非线性**的，因为色调H用角度（0°~360°）表示，采用了极坐标形式。HSV更符合人眼感知，但不是线性空间。

**13. 答案：C**

**（1）人脸识别完整流程（4分）：**

```

│ 1. 图像采集    │  采集人脸图像（正面、光照均匀）
```

**（2）PCA算法的计算步骤（4分）：**

**Step 1：数据准备与中心化**
- 输入：N个人脸图像，每张展开为d维向量（如64×64=4096维）
- 组成数据矩阵 X = [x₁, x₂, ..., xₙ]，大小为 d×N
- 计算均值向量 μ = (1/N)Σxᵢ
- 中心化：每个样本减去均值 φᵢ = xᵢ - μ
- 得到中心化矩阵 Φ = [φ₁, φ₂, ..., φₙ]

**Step 2：计算协方差矩阵**
- 协方差矩阵 C = (1/N) × Φ × Φᵀ
- C的大小为 d×d（如4096×4096），直接计算非常耗时

**Step 3：特征值分解（PCA的关键技巧）**
- 由于d远大于N，直接求C的特征向量计算量太大
- 技巧：先求小矩阵 L = Φᵀ × Φ（大小为N×N），求L的特征值λᵢ和特征向量vᵢ
- 再由 C的特征向量 uᵢ = Φ × vᵢ 得到原空间的特征向量
- 将特征值从大到小排序：λ₁ ≥ λ₂ ≥ ... ≥ λₙ

**Step 4：选择主成分**
- 选取前k个最大特征值对应的特征向量 u₁, u₂, ..., uₖ
- 选择标准：保留的方差比 ≥ 阈值（如95%），即 Σᵢ₌₁ᵏ λᵢ / Σᵢ₌₁ᴺ λᵢ ≥ 0.95
- 这k个特征向量就是"特征脸"（Eigenface）

**Step 5：降维投影**
- 投影矩阵 W = [u₁, u₂, ..., uₖ]，大小为 d×k
- 对任意人脸x，降维后的特征向量 y = Wᵀ × (x - μ)
- y的维度从d降到k（如从4096降到50），完成降维

**得分要点**：
- 步骤完整（5步）：2分
- 每步有具体公式和解释：2分

---

**（3）局限性与改进（2分）：**

**局限性：**

1. **对光照敏感**：PCA基于像素灰度值，光照变化会显著影响识别结果。同一人在不同光照下的差异可能大于不同人之间的差异。
2. **对姿态敏感**：PCA假设人脸是正面且对齐的，姿态变化（侧脸、低头等）会导致识别率下降。
3. **线性假设**：PCA是线性降维方法，无法捕捉人脸的非线性变化（如表情变化、遮挡等）。
4. **需要类别标签用于训练**：虽然PCA本身无监督，但识别阶段需要标签信息。
5. **计算量大**：当图像尺寸大、训练样本多时，协方差矩阵的特征值分解计算量很大。

**改进方向：**

1. **光照预处理**：在PCA之前加入光照归一化步骤（如直方图均衡化、商图像方法）。
2. **使用LDA替代或补充PCA**：LDA利用类别信息，能找到更有判别力的投影方向（Fisherface方法）。
3. **使用核PCA（Kernel PCA）**：通过核函数将数据映射到高维空间，处理非线性问题。
4. **使用深度学习方法**：如FaceNet、ArcFace等基于CNN的人脸识别方法，能自动学习鲁棒的人脸特征。
5. **数据增强**：通过旋转、翻转、加噪声等方式扩充训练集，提高模型的泛化能力。

**得分要点**：
- 列举至少3个局限性：1分
- 提出合理的改进方向：1分

### 计算题3：边缘检测——Sobel算子

**（1）Sobel算子系数2的作用：**

- Sobel算子在中心行/列的权重为2，目的是对边缘方向上的像素进行**加权平均**，增强边缘检测的抗噪能力。
- 权重2使得与边缘方向平行的像素对梯度计算的贡献更大，相当于在计算梯度之前先做了一次平滑，减少了噪声的影响。
- Prewitt算子的对应位置系数为1（等权重），没有这种加权平滑效果，因此抗噪性能不如Sobel。
- **总结**：Sobel = Prewitt + 加权平滑，系数2提高了抗噪性。

**（3）可分离卷积的乘法次数：**

**第一步**：用 k×1 核对图像逐列卷积
- 输出大小：508 × 512（行方向valid，列方向保持）
- 乘法次数：508 × 512 × 5 = 1,300,480

- 输出大小：508 × 508

**（2）HSV颜色空间分割红色区域（3分）：**

**提取红色区域的方法：**

在HSV颜色空间中，红色的H（色调）值分布在两个区间：
- H ∈ [0°, 10°] ∪ [170°, 180°]（OpenCV中H映射到[0,180]，即 H ∈ [0, 10] ∪ [170, 180]）
- S ∈ [50, 255]（饱和度较高，排除灰白色区域）
- V ∈ [50, 255]（明度适中，排除黑色区域）

**操作步骤：**
1. 将BGR图像转换为HSV图像（cv2.cvtColor）
2. 使用cv2.inRange设定红色的HSV阈值范围
3. 对红色H范围的两段分别做掩码，再进行逻辑或合并
4. 对掩码进行形态学操作（开运算去噪、闭运算填充孔洞）

**为什么HSV比RGB更适合颜色分割：**

| 特性 | RGB | HSV |
|------|-----|-----|
| 颜色与亮度 | 耦合在一起 | 分离（H+V独立） |
| 光照鲁棒性 | 差（光照变化影响三个通道） | 好（H和S对光照相对鲁棒） |
| 颜色表达 | 不直观 | 直观（H直接对应颜色种类） |
| 阈值设定 | 困难（需三个通道联合约束） | 简单（主要约束H通道） |

**核心原因**：HSV将颜色信息（H、S）与亮度信息（V）分离，使得在光照变化条件下，仅通过H和S通道就能稳定地识别颜色，而RGB的三个通道都受光照影响，难以设定稳定的颜色阈值。

**（2）协方差矩阵（3分）：**

C = (1/N) × Φ × Φᵀ = (1/3) × Φ × Φᵀ

先计算 Φ × Φᵀ（2×2矩阵）：

**（3）特征值和特征向量（3分）：**

λ(λ - 16/3) = 0

**求特征向量：**

对于 λ₁ = 16/3：
(C - λ₁I)v = 0
[8/3 - 16/3,  8/3    ] [v₁] = [0]
[8/3,         8/3 - 16/3] [v₂]   [0]

### 基于SIFT的图像匹配与拼接流程（15分）

---

**（1）SIFT算法的四个主要步骤（6分）：**

**Step 1：尺度空间极值检测（Detecting scale-space extrema）**

- **输入**：原始图像
- **处理过程**：
  1. 构建**高斯尺度空间**：对图像用不同尺度σ的高斯函数进行平滑，形成多组（octave）多层（scale）的图像金字塔。每组内σ按k倍递增：σ, kσ, k²σ, ...
  2. 构建**DoG（高斯差分）尺度空间**：相邻尺度的高斯图像相减，DoG(x,y,σ) = G(x,y,kσ) - G(x,y,σ)，近似LoG（尺度归一化拉普拉斯算子）
  3. **极值检测**：在DoG尺度空间中，每个像素与其同尺度8邻域 + 相邻尺度9×2=18个像素共26个邻居比较，若为局部极大值或极小值，则该位置为候选关键点
- **输出**：候选关键点的位置（x, y, σ）

**Step 2：关键点定位（Keypoint localization）**

- **输入**：候选关键点
- **处理过程**：
  1. **精确定位**：使用三维二次函数拟合DoG函数，将关键点位置精确到亚像素精度
  2. **低对比度点剔除**：若拟合后的DoG值低于阈值（如0.03），则丢弃该关键点（对比度太低，不稳定）
  3. **边缘响应剔除**：计算关键点处DoG函数的Hessian矩阵，通过主曲率比值判断是否为边缘响应。若主曲率比值过大（>10），则为边缘点，丢弃
- **输出**：精确定位后的稳定关键点（位置、尺度、DoG响应值）

**Step 3：方向赋值（Orientation assignment）**

- **输入**：稳定关键点及其尺度σ
- **处理过程**：
  1. 在关键点的邻域（半径1.5σ的窗口）内，计算每个像素的梯度幅值和方向
  2. 使用高斯加权（σ = 1.5σ_keypoint）对梯度幅值进行加权
  3. 构建**梯度方向直方图**：36个bin（每bin 10°），统计各方向的加权梯度幅值之和
  4. 直方图的**主峰方向**（最高bin的方向）作为关键点的主方向；若有超过主峰80%的次峰，则为该关键点创建多个关键点（不同方向），增强匹配的鲁棒性
- **输出**：具有位置、尺度和方向的关键点（每个关键点最多可有多个方向）

**Step 4：关键点描述子生成（Keypoint descriptor）**

- **输入**：具有位置、尺度、方向的关键点
- **处理过程**：
  1. 在关键点周围取16×16的邻域窗口
  2. 将窗口旋转到关键点的主方向（实现旋转不变性）
  3. 将16×16窗口划分为4×4=16个子区域（每个4×4像素）
  4. 在每个子区域内统计8个方向的梯度方向直方图（加权累加）
  5. 将16个子区域的8维直方图拼接，得到16×8=**128维**的特征向量
  6. 对128维向量进行归一化（L2范数归一化），消除光照变化的影响
  7. 截断大于0.2的分量，再次归一化（抑制非线性光照的影响）
- **输出**：128维的SIFT描述子向量

---

**（2）特征匹配过程（4分）：**

**Step 1：距离计算**
- 对图像A中的每个SIFT描述子 dAᵢ，计算与图像B中所有SIFT描述子 dBⱼ 的**欧氏距离**：dist(i,j) = ||dAᵢ - dBⱼ||₂

**Step 2：最近邻搜索**
- 对图像A中的每个描述子，找到图像B中距离最近的描述子（最近邻）和距离次近的描述子（次近邻）

**Step 3：NNDR比率筛选**
- 计算最近邻与次近邻的距离比值：ratio = dist_nearest / dist_second_nearest
- 若 ratio < 阈值（通常取0.4~0.8），则认为该匹配是正确的
- **原理**：正确匹配的最近邻距离远小于次近邻距离（ratio小），而错误匹配的最近邻和次近邻距离相近（ratio接近1）
- **阈值选择**：阈值越小，匹配数量越少但准确率越高；阈值越大，匹配数量越多但可能包含错误匹配

**Step 4：对称性检验（可选）**
- 从A到B的匹配和从B到A的匹配应该一致（双向一致性检查），只保留双向匹配一致的点对

**Step 5：RANSAC外点剔除（可选）**
- 使用RANSAC算法进一步剔除错误匹配（几何一致性检验）

---

**（3）从匹配点对到最终拼接图像的流程（5分）：**

**Step 1：单应性矩阵估计**
- 从匹配点对中至少选取4对点，使用**DLT（直接线性变换）**算法估计单应性矩阵H（3×3矩阵）
- H描述了两幅图像之间的几何变换关系：[x', y', 1]ᵀ ~ H × [x, y, 1]ᵀ
- 为提高鲁棒性，使用**RANSAC**算法：随机采样4对点估计H，计算所有匹配点的投影误差，保留内点（误差<阈值），迭代多次选择内点最多的H
- 输出：最优单应性矩阵H和内点集合

**Step 2：图像变换**
- 选定一幅图像为参考图（如左图），将另一幅图像（右图）通过H变换到参考图的坐标系中
- 对右图的每个像素(x, y)，计算其在参考图坐标系中的位置：(x', y') = H × (x, y, 1)ᵀ
- 由于变换后的坐标可能不是整数，需要进行**插值**（双线性插值或最近邻插值）
- 确定拼接后图像的画布大小（取两幅图像的最小/最大坐标范围）

**Step 3：图像融合**
- **直接拼接**：在重叠区域，简单选择其中一幅图像的像素。缺点：接缝处可能有明显的亮度差异和几何错位。
- **加权融合（渐入渐出）**：在重叠区域，根据像素到边界的距离进行加权混合：I_blend = α·I_left + (1-α)·I_right，其中α从1渐变到0。消除接缝效应。
- **多频段融合（Laplacian金字塔融合）**：将图像分解为不同频率的金字塔层，在低频部分大范围融合（避免色差），在高频部分小范围融合（保留细节）。效果最好但计算量大。

**Step 4：后处理**
- 裁剪拼接图像的黑色边界（无效区域）
- 可选：进行全局亮度均衡化，消除两幅图像的光照差异

**拼接流程总结：**
```
匹配点对 → RANSAC估计H → 图像变换+插值 → 重叠区域融合 → 裁剪输出
```
