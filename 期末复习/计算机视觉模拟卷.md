## 一、选择题答案与解析

**1. 答案：B**

解析：
- A错误：增大光圈（减小f值）会**减小**景深，使背景更加模糊，而非增大景深。
- **B正确**：根据成像公式1/f = 1/u + 1/v，焦距f越长，在相同物距u下像距v越大，成像越大（即放大倍率越高）。
- C错误：光圈大小不仅影响进光量，还直接影响景深——光圈越大景深越浅。
- D错误：缩小光圈（增大f值）会减少进光量，使图像亮度降低，而非增大。

**2. 答案：C**

解析：
- A错误：这是对杆体细胞的描述，不是锥体细胞。锥体细胞负责明视觉和颜色感知。
- B错误：杆体细胞主要分布在视网膜**周边**区域，中央凹处几乎只有锥体细胞。
- **C正确**：杆体细胞（约1.2亿个）对光的敏感度很高，负责暗环境下的视觉（夜视），但不感知颜色差异。
- D错误：两者分布不均匀——中央凹以锥体细胞为主，周边以杆体细胞为主，功能也不同。

**3. 答案：A**

解析：
- **A正确**：BRDF（Bidirectional Reflectance Distribution Function）的定义为 fr(ωi, ωr) = dLr(ωr) / (Li(ωi)cosθi dωi)，即出射辐射亮度与入射辐照度之比，描述了光线在物体表面的反射特性。
- B错误：BRDF同时依赖于入射方向ωi和出射方向ωr。
- C错误：BRDF的单位是sr⁻¹（每球面度），不是瓦特。
- D错误：BRDF描述的是物体表面材质的反射特性，与光源无关。

**4. 答案：B**

解析：
- A错误：显示器不是成像过程的基本要素。
- **B正确**：成像三要素为：①照明（光源特性）、②场景/物体（反射/透射特性）、③传感器（成像设备/感光元件）。
- C错误：镜头、快门、光圈是相机的组成部分，不是成像三要素。
- D错误：像素、分辨率、灰度级是图像的属性参数，不是成像三要素。

**5. 答案：C**

解析：
- A正确：深度学习（CNN等）在ImageNet分类、COCO检测等任务上取得突破性成果。
- B正确：GPT-4V、LLaVA等视觉-语言多模态大模型是当前热点。
- **C不正确**：计算机视觉仍面临许多未解决的挑战，如开放世界识别、3D理解、鲁棒性等，远未"完全解决"。
- D正确：三维视觉、视频理解、具身智能（embodied AI）等是新兴方向。

**6. 答案：C**

解析：
- A错误：CMYK是**减色模型**，用于印刷；RGB是加色模型，用于显示器。
- B错误：RGB的三个分量与人眼感知的色调、饱和度、亮度**不直接对应**，这是HSI空间的优势。
- **C正确**：HSI（色调Hue、饱和度Saturation、强度Intensity）颜色空间将亮度与色度分离，更符合人眼的感知方式，在图像处理中广泛使用。
- D错误：YUV中Y表示**亮度**，U和V表示**色度**分量，正好说反了。

**7. 答案：B**

解析：
- A选项：512×512×8bit = 512×512×1 byte = 262144 bytes = 256KB，数值上正确，但"约为"的说法不够精确。
- **B正确**：8比特灰度级意味着每个像素有2⁸ = 256个可能的灰度值（0~255）。这是灰度级的基本定义。
- C错误：空间分辨率（像素数）和灰度级（每像素比特数）是两个独立的参数，之间没有必然的因果关系。
- D错误：图像总存储空间 = 空间分辨率（像素数）× 灰度级（比特/像素），两者直接相关。

**8. 答案：B**

解析：
- A错误：点操作只考虑**当前像素**本身的灰度值，不涉及邻域信息；考虑邻域的操作是局部操作。
- **B正确**：对数变换 s = c·log(1+r) 的特点是扩展低灰度值区域（暗区域）的动态范围，压缩高灰度值区域（亮区域）的动态范围。
- C错误：γ>1的幂次变换（s = crᵞ）会压缩整体灰度范围，使图像变暗；γ<1时才会扩展暗区域。
- D错误：分段线性变换可以通过设计不同的折线函数，**灵活地突出**任意感兴趣的灰度区间。

**9. 答案：B**

解析：
- 直接进行二维3×3卷积：每个像素需要9次乘法和8次加法，总复杂度为O(M×N×9)。
- 3×3均值滤波核可以分离为一个3×1列向量和一个1×3行向量：先对每行做1×3卷积（每像素3次乘法），再对结果做3×1卷积（每像素3次乘法），总复杂度为O(M×N×6)。
- 因此可分离卷积的复杂度约为O(M×N×6)，即**B正确**。
- 更一般地，对于k×k的可分离核，直接二维卷积复杂度O(MNk²)，可分离卷积复杂度O(2MNk)。

**10. 答案：B**

解析：
- A错误：理想低通滤波器在截止频率处是**陡峭截止**（无过渡），**会产生严重的振铃效应**。
- **B正确**：巴特沃斯低通滤波器在通带和阻带之间有平滑的过渡带，振铃效应比理想滤波器轻得多。
- C错误：高斯低通滤波器的过渡带**最宽**（最平滑），频率选择性最差，但**不产生振铃效应**。
- D错误：高通滤波器用于**锐化**图像、增强边缘，低通滤波器才用于平滑和去噪。

**11. 答案：B**

解析：
- 膨胀操作的定义：用结构元素扫描图像，只要结构元素覆盖区域内有一个白色像素，中心位置即变为白色。
- 对一个孤立白色像素点，用3×3全1结构元素进行膨胀时，结构元素中心放在该像素上，覆盖了该像素及其8邻域所有位置，因此这些位置全部变为白色。
- **B正确**：该像素点及其8邻域内所有像素变为白色。
- A错误：膨胀后不再是孤立点。
- C错误：3×3全1结构元素覆盖的是8邻域（含对角），不是4邻域。
- D错误：膨胀操作不会使前景像素变黑。

**12. 答案：C**

解析：
- A正确：K-means需要预设K值。
- B正确：不同的初始聚类中心可能导致不同的最终结果，容易陷入局部最优。

**（3）腐蚀与膨胀的物理意义及开运算效果：**

- **腐蚀**：用结构元素"探测"图像，只有当结构元素完全被前景区域包含时，该位置才保留为前景。效果是使前景区域缩小、消除小的孤立噪声点、分离粘连的物体。
- **膨胀**：用结构元素扫描图像，只要结构元素与前景区域有交集，该位置就变为前景。效果是使前景区域扩大、填补前景中的小孔洞、连接邻近的物体。
- **开运算**（先腐蚀后膨胀）：可以消除小的孤立噪声点和毛刺，同时保持较大物体的形状和大小基本不变。开运算具有"平滑轮廓、断开窄连接、去除小突起"的效果。

## 三、综合题答案与解析

### 基于PCA的人脸识别系统

**（1）基于PCA的人脸识别完整流程：**

① **图像采集**：获取人脸图像（如摄像头拍摄或从人脸数据库读取）。

② **预处理**：灰度化 → 几何归一化（统一尺寸，如裁剪为92×112）→ 光照归一化（直方图均衡化，消除光照影响）。

③ **数据向量化**：将每张二维人脸图像按行展开为一维列向量（如92×112=10304维），所有训练样本组成数据矩阵X。

④ **PCA降维（特征脸提取）**：
- 中心化：每个样本减去所有人脸的均值脸μ
- 计算协方差矩阵：C = (1/n)·XᵀX
- 特征值分解：求C的特征值λ和特征向量w
- 选取前k个最大特征值对应的特征向量，组成投影矩阵W（这些特征向量就是"特征脸/Eigenface"）
- 降维：Y = WᵀX，将高维人脸投影到低维子空间

⑤ **训练分类器/建立数据库**：将训练集中每个人的PCA特征（Y的列向量）存入数据库。

⑥ **识别**：对输入的测试人脸做同样的PCA投影，得到特征向量，与数据库中所有已知人脸计算欧氏距离，距离最近的即为识别结果。

**（2）PCA算法具体步骤：**

| 步骤 | 操作 | 作用 |
|------|------|------|
| ① 数据中心化 | 每个样本减去均值μ | 使数据零均值化，消除偏移 |
| ② 计算协方差矩阵 | C = (1/n)·XᵀX | 描述各维度之间的相关性 |
| ③ 特征值分解 | 求C的特征值λ和特征向量w | 找到数据变化最大的方向 |
| ④ 排序选主成分 | 取前k个最大特征值的特征向量组成W | 保留最重要的信息 |
| ⑤ 降维投影 | Y = WᵀX | 将高维数据降到k维 |

**（3）局限性与改进：**

- **局限**：PCA是无监督的，不利用类别信息，提取的特征可能不是最具判别力的；对光照、表情、姿态变化敏感。
- **改进**：① 用LDA替代PCA（有监督，最大化类间/类内散度比）；② 用深度学习方法（如FaceNet、ArcFace）学习更具判别性的特征；③ 结合LBP等局部特征增强鲁棒性。

解析：
- A错误：数字说反了。人眼约有**1.2亿个杆体细胞**和**600万个锥体细胞**。
- **B正确**：锥体细胞（约600万个）集中在视网膜中央凹区域，负责明亮环境下的高分辨率视觉和颜色感知。

- A错误：辐射度量（如辐射通量、辐射强度）描述的是**客观的物理辐射能量**，不涉及人眼感知。

### 计算题3：边缘检测（Sobel算子）

**原始图像矩阵：**
```
  0   0   0   0   0
  0  50  50  50   0
  0  50 100  50   0
  0  50  50  50   0
  0   0   0   0   0
```

**（1）计算像素(2,2)处的梯度Gₓ和Gᵧ：**

像素(2,2)的3×3邻域为：
```
50   50   50
50  100   50
50   50   50
```

**水平梯度Gₓ**（使用Sobel Gₓ算子卷积）：

Gₓ = (-1)×50 + 0×50 + 1×50 + (-2)×50 + 0×100 + 2×50 + (-1)×50 + 0×50 + 1×50
= -50 + 0 + 50 + (-100) + 0 + 100 + (-50) + 0 + 50
= **0**

**垂直梯度Gᵧ**（使用Sobel Gᵧ算子卷积）：

Gᵧ = (-1)×50 + (-2)×50 + (-1)×50 + 0×50 + 0×100 + 0×50 + 1×50 + 2×50 + 1×50
= -50 + (-100) + (-50) + 0 + 0 + 0 + 50 + 100 + 50
= **0**

**（2）计算梯度幅值和方向：**

梯度幅值：
|G| = √(Gₓ² + Gᵧ²) = √(0² + 0²) = **0**

梯度方向：
θ = arctan(Gᵧ/Gₓ) = arctan(0/0) = **未定义**（梯度为零向量，方向无意义）

**（3）分析：该像素是否为"非边缘"像素？**

是的，梯度计算结果**正确地反映了像素(2,2)是非边缘像素**。

原因分析：
- 像素(2,2)的灰度值为100，虽然它是图像中的最高值，但其**周围8邻域的灰度值完全对称**——上、下、左、右四个方向都是50，四个对角方向也都是50。
- Sobel算子Gₓ检测的是**水平方向**的灰度变化：左侧(-1×50-2×50-1×50 = -200)与右侧(1×50+2×50+1×50 = 200)完全对称，因此Gₓ = 0。
- Sobel算子Gᵧ检测的是**垂直方向**的灰度变化：上方(-1×50-2×50-1×50 = -200)与下方(1×50+2×50+1×50 = 200)完全对称，因此Gᵧ = 0。
- 这说明像素(2,2)虽然灰度值高，但它处于一个**对称的局部极值点**（类似于一个"斑点"中心），在各个方向上灰度变化的梯度相互抵消，**不是边缘**。
- 边缘的定义是灰度在某一方向上发生**单向突变**（如从暗到亮），而像素(2,2)处的灰度突变是对称的（四面都是从100降到50），所以梯度为零。
- 如果要检测这种"斑点"特征，应使用二阶微分算子（如拉普拉斯算子），它对这种对称的灰度极值点会有非零响应。

### 计算题4：卷积复杂度计算

**（1）不可分离卷积的计算复杂度：**

- 对于k×k的不可分离卷积核，每个像素需要进行：
  - 乘法次数：**k²次**（卷积核的每个元素与对应像素相乘）
  - 加法次数：**k²-1次**（将k²个乘积相加需要k²-1次加法）
- 图像共有M×N个像素，因此：
  - 总乘法次数 = M × N × k²
  - 总加法次数 = M × N × (k²-1)
  - 总计算复杂度 = **O(M × N × k²)**

**（2）可分离卷积的计算复杂度：**

- 对于可分离的k×k卷积核（分解为k×1列向量和1×k行向量），每个像素需要进行：
  - 第一步（用1×k行向量卷积）：每像素k次乘法，k-1次加法
  - 第二步（用k×1列向量卷积）：每像素k次乘法，k-1次加法
  - 总计：每像素 **2k次乘法**，**2(k-1)次加法**
- 图像共有M×N个像素，因此：
  - 总乘法次数 = M × N × 2k
  - 总加法次数 = M × N × 2(k-1)
  - 总计算复杂度 = **O(M × N × 2k)**

**（3）当M=N=1024，k=7时的具体计算：**

- **不可分离卷积**：
  - 总乘法次数 = 1024 × 1024 × 7² = 1048576 × 49 = **51,380,224次**

- **可分离卷积**：
  - 总乘法次数 = 1024 × 1024 × 2 × 7 = 1048576 × 14 = **14,680,064次**

- **节省的计算量**：
  - 节省的乘法次数 = 51380224 - 14680064 = 36700160次
  - 节省百分比 = (36700160 / 51380224) × 100% ≈ **71.4%**
  - 即可分离卷积的计算量约为直接卷积的 **28.6%**（即14/49）

- 一般结论：对于k×k可分离核，可分离方法的计算量是直接方法的 **2k/k² = 2/k**。当k=7时，2/7 ≈ 28.6%，节省约71.4%的计算量。k越大，可分离方法的优势越明显。

## 三、综合题答案与解析

### 交通标志定位与识别系统

**（1）交通标志检测与识别的完整流程：**

① **图像采集**：通过车载摄像头获取道路场景图像，或从视频中逐帧读取。

② **预处理**：尺寸归一化 → 去噪（高斯/中值滤波）→ 颜色空间转换（RGB→HSV，便于颜色分割）→ 亮度校正（直方图均衡化）。

③ **候选区域检测（颜色筛选）**：在HSV空间中，根据交通标志常见颜色（红、蓝、黄）进行阈值分割，提取可能包含标志的区域。再用形态学操作（膨胀连接、腐蚀去噪）修正区域边界。

④ **目标定位（形状筛选）**：对候选区域找轮廓，用多边形拟合判断形状（圆形、三角形、矩形），计算外接矩形，过滤面积过小、长宽比异常的区域。可用Hough圆检测辅助筛选圆形标志。

⑤ **特征提取**：对定位到的标志区域提取特征，如颜色直方图、HOG特征（梯度方向直方图）、SIFT/ORB局部特征、或CNN深层特征。

⑥ **分类识别**：使用SVM、K-NN或CNN分类器对特征向量分类，输出交通标志类别（限速、禁止通行、停车、警告等）。

⑦ **后处理**：合并重叠检测框，删除低置信度结果，输出最终识别结果并绘制边界框和类别标签。

**（2）颜色和形状特征筛选方法：**

- **颜色筛选**：将图像从RGB转到HSV空间，设定H（色调）的阈值范围提取红色（H≈0°或180°）、蓝色（H≈120°）、黄色（H≈30°）区域。利用S通道排除低饱和度的非彩色区域。对颜色分割结果做形态学开运算去噪、闭运算填补空洞。

- **形状筛选**：用Canny边缘检测获取边缘 → findContours找轮廓 → approxPolyDP多边形近似 → 根据顶点数判断形状（3顶点=三角形警告标志，4顶点=矩形指示标志，圆形用圆形度判断）→ 用boundingRect计算外接矩形，根据面积和长宽比过滤。

**（3）特征提取方法和分类器：**

**特征提取方法：**
- **颜色直方图**：统计标志区域内各颜色通道的分布
- **HOG（梯度方向直方图）**：统计局部区域的梯度方向和幅值分布，对形状特征描述能力强
- **LBP（局部二值模式）**：描述纹理特征，对光照变化鲁棒
- **SIFT/ORB**：提取局部关键点和描述子，适合有旋转和尺度变化的场景
- **CNN特征**：用预训练的CNN（如ResNet）提取深层语义特征

**分类器：**
- **SVM**：适合中小规模数据集，分类精度高
- **K-NN**：简单直观，适合类别数较少的场景
- **随机森林**：集成学习方法，抗过拟合能力强
- **CNN/ResNet**：端到端深度学习分类，精度最高但需要大量训练数据
