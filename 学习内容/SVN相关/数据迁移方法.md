### 一、图形化迁移向导（简单高效）

此方法使用VisualSVN Server内置的迁移向导，适合大多数场景。步骤基于引用[1]：

1. **停止服务并准备**  
    通知所有用户停止SVN操作，在旧服务器上停止VisualSVN Server服务（通过服务管理器或命令行执行 `net stop VisualSVN`）_4_。
    
2. **启动迁移向导**  
    在目标服务器（新服务器）上打开VisualSVN Server Manager，导航到“Tools”菜单，选择“Migration Wizard”。
    
3. **选择迁移类型**  
    在向导中选择“迁移到新的服务器”（Migrate to a new server）_1_。
    
4. **配置服务器信息**
    
    - 输入源服务器（旧服务器）的IP地址、管理员用户名和密码。
        
    - 输入目标服务器（新服务器）的IP地址和凭据。
        
    - 向导会自动检查目标服务器的兼容性（如VisualSVN版本是否匹配），并给出调整建议_1_。
        
5. **选择迁移内容**  
    勾选要迁移的仓库（repositories）和用户组（groups），支持全选或部分迁移。
    
6. **执行迁移**  
    点击“Start Migration”，向导会自动复制数据并显示进度条。完成后，验证新服务器上的仓库和用户组是否完整_1_。
    
7. **迁移后设置权限**  
    由于权限配置可能丢失，需在VisualSVN Server Manager中重新为每个仓库设置访问权限（右键点击仓库 → Properties → Security）_3_。
    

**注意事项**：

- 确保新旧服务器运行相同版本的VisualSVN Server（如3.3.1）_4_。
    
- 如果源服务器使用Windows认证，目标服务器必须加入相同域_2_。
    
- 迁移期间避免网络中断。
    

### 二、手动迁移（命令行方式）

此方法通过svnadmin工具操作，适合大规模仓库或自定义需求。步骤基于引用[2][4][5]：

1. **停止服务并备份**  
    在旧服务器停止VisualSVN Server服务，并备份 %VISUALSVN_SERVER目录。
    
2. **导出仓库数据（Dump）**  
    在旧服务器打开PowerShell或命令提示符，使用 `svnadmin dump` 命令：
    svnadmin.exe dump "C:\Repositories\your_repo" > "C:\backup\repo.dump"
    将生成的dump文件复制到新服务器。

3. **导入仓库数据（Load）**  
    在新服务器上：
    - 创建空仓库：`svnadmin.exe create "C:\Repositories\new_repo"`
        
    - 加载数据：`svnadmin.exe load "C:\Repositories\new_repo" < "C:\backup\repo.dump"`_5_
        
4. **迁移用户和组数据**
    
    - 复制旧服务器 `%VISUALSVN_SERVER%data` 目录下的所有文件（如 `htpasswd`, `groups.xml`, `authz`）到新服务器对应目录，覆盖现有文件_2_。
        
    - 若使用Windows认证，跳过文件复制，但确保新服务器在相同域_2_。
        
5. **启动服务并验证**  
    启动新服务器的VisualSVN服务（`net start VisualSVN`），在VisualSVN Server Manager中检查仓库列表和用户组。重新设置仓库权限（方法同向导步骤7）_3_。
    

**通用注意事项**：

- **兼容性**：新旧服务器操作系统和VisualSVN版本需一致（如Windows Server 2012 R2 + VisualSVN 3.3.1）_4_。
    
- **性能**：大型仓库迁移时，dump/load过程可能耗时；建议在低峰期操作。
    
- **测试**：迁移后执行SVN检出（checkout）和提交（commit）测试。
    
- **备份**：始终保留旧服务器数据直至验证成功_5_。
