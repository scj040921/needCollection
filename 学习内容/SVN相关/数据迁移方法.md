### 一、图形化迁移向导（简单高效）

此方法使用VisualSVN Server内置的迁移向导，适合大多数场景。步骤基于引用[1]：

1. **停止服务并准备**  
    通知所有用户停止SVN操作，在旧服务器上停止VisualSVN Server服务（通过服务管理器或命令行执行 `net stop VisualSVN`）_4_。
    
2. **启动迁移向导**  
    在目标服务器（新服务器）上打开VisualSVN Server Manager，导航到“Tools”菜单，选择“Migration Wizard”。
    
3. **选择迁移类型**  
    在向导中选择“迁移到新的服务器”（Migrate to a new server）_1_。
    
4. **配置服务器信息**
    
    - 输入源服务器（旧服务器）的IP地址、管理员用户名和密码。
        
    - 输入目标服务器（新服务器）的IP地址和凭据。
        
    - 向导会自动检查目标服务器的兼容性（如VisualSVN版本是否匹配），并给出调整建议_1_。
        
5. **选择迁移内容**  
    勾选要迁移的仓库（repositories）和用户组（groups），支持全选或部分迁移。
    
6. **执行迁移**  
    点击“Start Migration”，向导会自动复制数据并显示进度条。完成后，验证新服务器上的仓库和用户组是否完整_1_。
    
7. **迁移后设置权限**  
    由于权限配置可能丢失，需在VisualSVN Server Manager中重新为每个仓库设置访问权限（右键点击仓库 → Properties → Security）_3_。
    

**注意事项**：

- 确保新旧服务器运行相同版本的VisualSVN Server（如3.3.1）_4_。
    
- 如果源服务器使用Windows认证，目标服务器必须加入相同域_2_。
    
- 迁移期间避免网络中断。
    

### 二、手动迁移

#### 1. **导出仓库备份文件**
- **在旧服务器（低版本）操作**：
    1. 打开 **VisualSVN Server Manager** → 停止服务（右键服务器名 → **Stop**）
    2. 右键点击需迁移的仓库 → **Export Backup**。
    3. 选择保存路径→ 设置备份密码（可选）→ 完成。
#### 2. **迁移备份文件到新服务器**
- 将生成的 `.svnbackup` 文件复制到新服务器（高版本）的任意目录。
#### 3. **导入仓库到新服务器**
- **在新服务器（高版本）操作**：
    1. 打开 **VisualSVN Server Manager**。
    2. 右键点击 **Repositories** → **Import Repository**。
    3. 选择备份文件路径
#### 4. **迁移用户与权限**
- **复制配置文件**（需手动覆盖）：
    1. 从旧服务器的 `data` 目录复制：
        - `htpasswd`（用户密码）
        - `authz`（权限规则）
        - `groups.xml`（用户组）
    2. 粘贴到新服务器的 `data` 目录（默认 `C:\Program Files\VisualSVN Server\data`）→ **覆盖文件**_1_。
        
- **重新加载配置**：
    
    - 在 VisualSVN Server Manager 中右键点击 **Users** 或 **Groups** → **Refresh**。
        

#### 5. **验证权限兼容性**

- 打开仓库的 **Properties** → **Security**：
    
    1. 检查用户/组权限是否完整。
        
    2. 若权限丢失（高版本权限模型可能升级），手动重新添加用户/组_1_。
