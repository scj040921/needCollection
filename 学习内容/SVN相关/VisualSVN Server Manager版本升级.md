# 说明
## 1.版本选择：
- 推荐最新的5.4.6版本，更好用也更稳定
- 但是当前使用版本为2.5.6，比较古老跨服过大可能出现一些问题
## 2.推荐更新方法
1. 直接升级（官方推荐）
   *风险比较大，如果失败回滚复杂*
2. 为了避免跨度过大带来的问题，可以采用分段升级的方法（个人倾向于）
   *路径：2.5.6 → 3.5.6 → 4.3.3 → 5.4.5*
3. 并行部署
   *新建 5.4.5 服务器 + 迁移数据*
## 3.每种升级方法可能导致的问题
### 直接升级：
1. 仓库格式兼容性问题：2.5.6使用的FSFS格式版本较旧（可能是v2或v3），而5.4.5要求v6或更高
2. 用户认证方式丢失：
    - 2.5.6使用本地用户数据库，升级后可能需要重新配置 
    - Windows域认证配置可能丢失 
    - 密码加密算法变更导致密码失效
3. SSL证书失效：
    - 旧版本的自签名证书格式不兼容 
    - 证书绑定到旧版本的服务配置 
    - 证书存储位置变更
4. 权限配置丢失或混乱：
    - 权限配置文件格式变更 
    - 用户ID映射关系改变
    - 组策略配置不兼容
5. 端口冲突：
    - 新版本默认端口可能与旧版本不同 
    - Apache配置文件冲突 
    - 防火墙规则未更新
6. 客户端兼容性问题：
    - 新版本服务器要求客户端版本≥1.8 
    - 协议版本不匹配 
    - SSL/TLS版本要求提高
### 分段升级：
1. 中间版本的许可证问题
2. 配置重复清理
3. 升级耗时长
### 并行部署：
1. 权限配置丢失需要重新配置
2. 客户端需要重新复制
