# 搜索的基本概念
##  1. 采用某种策略，在所规定的约束条件下，从初始实时到目标状态构造一条代价较小的推理路径，使问题得到解决的计算过程称为搜索
# 搜索策略
## 搜索分类
- 盲目搜索就是无“**向导**”的搜索。树式盲目搜索就是穷举式搜索，即从初始节点出发，沿着连接边逐一考察各个节点，或者反向进行。
- 按搜索范围的扩展顺序的不同，搜索又可分为**广度优先搜索**（BFS）和**深度优先搜索**（DFS)
### BFS(广度优先搜索)
#### 基本概念
**广度优先搜索**（Breadth-First Search，简称BFS）是一种用于遍历或搜索树或图的经典算法。它从起始节点开始，**按层次遍历**所有节点，确保先访问所有相邻节点后再深入下一层。
#### 核心思想
1. **水波扩散原理**：
   像水波向外扩散一样，BFS总是先探索距离起点最近的的节点，然后逐步向外扩展。
2. **层次遍历**：
   - 从起点开始，先访问所有直接相邻的节点
   - 再访问这些相邻节点的相邻节点
   - 以此类推，直到遍历完所有可达节点
#### 算法流程
1. 初始化：将起始节点放入队列，标记为已访问
2. 循环执行：
   - 从队列头部取出一个节点
   - 检查该节点是否为目标节点
   - 如果是，返回结果
   - 如果不是，将该节点所有未访问的相邻节点加入队列尾部
   - 标记这些节点为已访问
3. 终止条件：队列为空或者找到目标节点
#### 关键特点

| 特点     | 说明                       |
| ------ | ------------------------ |
| 最短路径保证 | 在无权图中，第一次到达目标节点的路径一定是最短的 |
| 系统性    | 彻底搜索整张图，不依赖经验法则          |
| 避免重复   | 通过标记已访问节点防止循环            |
| 空间复杂度  | O(V),V为节点个数              |
| 时间复杂度  | O(V+E)，V为节点数，E为边数        |
#### 缺点：
盲目性较大，当目标节点距离起始节点较远时将会产生许多无用节点，搜索效率低。
### DFS（深度优先搜索）
#### 基本概念：
**深度优先搜索**（Depth-First Search，简称DFS）是一种用于遍历或搜索树或图的经典算法。它从起始节点开始，**沿着一条路径尽可能深入**，直到无法继续为止，然后回溯到上一个节点，继续探索其他路径。
#### 核心思想：
1. **走迷宫原理**：像一个人在迷宫中探索，总是选择一条路走到黑，直到遇到死路，才返回上一个岔路口，选择另一条路继续前进。
2. **深度优先**：
   - 从起点开始，选择一个未访问的相邻节点
   - 递归地深入该节点，继续选择未访问的相邻节点
   - 当当前节点没有未访问的相邻节点时，回溯到上一个节点
   - 重复这个过程，直到遍历完所有可达节点
#### 算法流程：
1. 初始化：选择起始节点标记为已访问
2. 递归执行：
   - 访问当前节点
   - 选择一个未访问的相邻节点
   - 递归地对该节点执行DFS
   - 如果没有未访问地相邻节点，回溯
3. 终止条件：所有节点都被访问或找到目标节点
#### 关键特点：

| 特点      | 说明                |
| ------- | ----------------- |
| 探索深度    | 有限探索一条路径到最深处      |
| 回溯机制    | 当无法继续深入时返回上一个节点   |
| 空间复杂度   | O(h),h为树的高度       |
| 时间复杂度   | O(V+E),V为节点数，E为边数 |
| 不保证最短路径 | ~                 |
#### 与BFS对比

| 特性    | DFS          | BFS          |
| ----- | ------------ | ------------ |
| 数据结构  | 栈            | 队列           |
| 遍历方式  | 深度优先         | 层次遍历         |
| 最短路径  | 不保证          | 保证最短         |
| 空间复杂度 | 低            | 高            |
| 使用场景  | 路径存在性问题、组合问题 | 最短路径问题、连通性问题 |
| 实现难度  | 简单           | 简单           |
### 启发式搜索
#### 启发式搜索基本概念：
**启发式搜索**（Heuristic Search）又称为**有信息搜索**（Informed Search），是一种利用问题的**启发信息**来引导搜索过程的智能算法。它通过评估函数对搜索空间中的节点进行排序，优先探索最有希望的路径，从而显著减少搜索范围，提高搜索效率。
#### 核心思想：
- 利用额外信息：不仅考虑已走过的路径，还预测到目标的距离
- 智能引导：优先探索最有希望的路径
- 效率优先：在保证最优性的同时，大幅减少无效搜索
#### 关键组成部分
1. **估价函数**(f(n))：是启发式搜索的核心，用于评估每个节点的希望值
f(n)=g(n)+h(n)

| 符号   | 含义    | 说明                 |
| ---- | ----- | ------------------ |
| f(n) | 总评估代价 | 从起点经过节点n到达目标的总代价估计 |
| g(n) | 实际代价  | 从起点到当前节点n的真实路径代价   |
| h(n) | 启发式代价 | 从当前节点n到目标节点的估计代价   |
2. **启发式函数**：
   启发式函数 h(n)h(n) 是算法的灵魂，它需要满足：
  - **可容性**（Admissibility）： h(n)≤h∗(n)h(n)≤h∗(n) ，即不会高估实际代价
  - **一致性**（Consistency）： h(n)≤c(n,n′)+h(n′)h(n)≤c(n,n′)+h(n′) ，满足三角不等式
#### 重要算法：
1. **A算法**：
   - **A算法**（A Algorithm）是一种经典的**启发式搜索算法**，它是‘A*’算法的前身和更一般化的形式。A算法通过估价函数来评估节点的"希望值"，从而指导搜索方向，提高搜索效率。
   - 启发式函数的约束条件：
     - **无约束**：启发式函数 h(n)h(n) 可以任意设计
     - 可以高估实际代价
     - 可以低估实际代价
     - 灵活性高，但可能找不到最优解
1. **’A*‘算法**：
   - **‘A*’算法**（A-Star Algorithm）是一种经典的**启发式搜索算法**，广泛应用于路径规划、游戏AI、机器人导航等领域。它结合了**Dijkstra算法**的全局最优性和**贪心算法**的高效性，能够在复杂的图结构中快速找到从起点到终点的最优路径。
   - 启发式函数的约束条件：
     - **必须满足可容性**： h(n)≤h∗(n)h(n)≤h∗(n)
       - h(n)h(n) ：启发式函数的估计值
       - h∗(n)h∗(n) ：从节点 nn 到目标的实际最小代价
     - **不能高估**实际代价
     - 保证最优解
