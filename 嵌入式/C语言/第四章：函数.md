# 1. 函数的基本概念
- 在C语言中，函数意味着功能模块。典型的C语言程序是由多个功能模块拼接组成，因此C语言被称为模块化语言。函数是封装的，对外提供的接口简洁，内部功能明确
1. **函数定义**
	- - 典型函数定义的语法如下：
		返回类型 函数名称(参数1, 参数2, ……)
		{
		    函数体
		}
	- 函数头：第一行称为函数头，或称函数特征表，是函数对外的接口
		1. 函数名称：命名规则与变量一致
		2. 参数列表：即黑箱的输入数据列表，一个函数可以有多个参数也可以没有参数
		3. 返回类型：即黑箱的输出数据类型，一个函数可以不返回数据，但最多只能返回一个数据。
	- 函数体：花括号内为函数体，是函数功能的内部实现
	- 语法：
		1. 当函数的参数列表为void时，表示该函数不需要任何参数
		2. 当函数的返回类型为void时，表示该函数不返回任何数据
		3. 关键字return表示退出函数。若函数头中规定有返回数据类型，则return需要携带一个类型与之匹配的数据；若函数头中规定返回类型为void，则return不需携带参数。
2. **函数调用**
	- 函数调用必须携带圆括号，里面依次放入函数定义中所规定的参数列表，参数的类型、个数、顺序都必须跟定义一致。
3. **函数声明**
	- 先声明再调用，顺序不可反
# 2. 重要概念
1. **形参实参**
	- 概念：
		- 函数调用中的参数，被称为实参
		- 函数定义中的参数，被称为形参
	- 实参与形参的关系
		- 实参与形参的类型和个数必须一一对应
		- 形参的值由实参初始化
		- 形参与实参位于不同的内存区域，彼此独立
2. **函数调用的流程**
	- 函数调用时，进程的上下文会切换到被调函数，当被调函数执行完成之后再切换回去
3. **局部变量与栈内存**
	- 局部变量概念：凡是被花括号包裹的的变量称为局部变量
	- 局部变量的特点：
		- 某一函数的局部变量存储在该函数特定的栈内存中
		- 局部变量旨在该函数内部可见，在该函数外部不可见
		- 当该函数退出后，局部变量所占的内存立即被系统回收，因此局部变量也称为临时变量
		- 函数的形参虽然不被花括号所包含，但依然属于该函数的局部变量
	- 占内存的特点：
		- 每当一个函数被调用时，系统将自动分配一段栈内存给该函数，用于存放其局部变量
		- 每当一个函数退出时，系统将自动回收其栈内存
		- 系统为函数分配栈内存时，遵从从上往下分配的规则
	- 技术要点：
		- 占内存相对而言是比较小的，不适合用来分配尺寸太大的变量
		- return之后不可再访问函数的局部变量，因此返回一个局部变量的地址通常是错误的
# 特殊函数
## 1.静态函数
- **背景知识**：普通函数都是跨文件可见的
- **静态函数**：只能在定义的文件内可见的函数
- **要点**：
	- 静态函数主要是为了缩小函数的可见范围，减少与其它文件中重名函数冲突的概率
	- 静态函数一般被定义在头文件中，然后被各个源文件包含
- **static关键字**：
	- 修饰函数：改变可见范围，由全局可见改为单文件可见
	- 修饰全局变量：改变其可见范围，由全局可见改为单文件可见
	- 修饰局部变量：改变其生命周期，由原本的函数结束就回收改为程序退出才回收
## 2.递归函数
- **概念**：如果一个函数内部包含了对自身的调用，则该函数称为递归函数
- **要点**：
	- 只有能被表达为递归的问题才能用递归函数解决
	- 递归函数必须有一个可直接退出的条件，否则会进入无限递归
	- 递归函数包含两个过程，一个逐渐递进的过程，和一个逐渐回归的过程
- 随着递归函数的层层深入，栈空间逐渐往下增长，很容易耗尽栈内存
- 层层递进时，问题规模会随之减小，减小到可直接退出的条件时，函数开始层层回归
## 3.回调函数
- **概念**：函数实现者不调用该函数，而由函数接口提供者间接调用的函数
- 应用开发者和内核开发者只要约定好回调函数的接口，即可各自开发进度互不影响
## 4.内联函数
- **使用场景**：
	- 当一个函数被频繁调用时
	- 当一个函数短小精悍时
- 只需在函数头前加一个inline即可，但是内联函数和普通函数有一个区别：其默认情况下不会生成可被连接的符号，因此必须与extern（全局）或static（局部）配合使用，来说明内联函数时全局符号还是局部符号。
- 一般会将内联函数定义在头文件中，并加上static声明，然后被各个源文件包含
- 优点是缩短了执行时间，缺点是程序体积变大
- 内联函数相当于直接拷贝编译后的指令到现场，而宏定义直接复制源代码
## 5.弱函数
- **作用**：给函数定义增加一层保护，防止函数定义重复
- **语法**：在函数头前加_attribute_((weak))
- 通过弱函数可以给用户提供一个默认的函数实现，但用户可以选择使用默认函数或对默认函数重新定义
# 字符串函数
## 1.查找字串：strstr()
- **功能**：在一个指定的字符串中找到一个子串
- 需要头文件：#include<string.h>
- 需要的参数一次是指定的字符串和需要查找的子串
- 若查找成功则返回指向子串的指针，否则返回空
## 2.查找字符：strchr()与strrchr()
- **功能**：在指定字符串内查找特定字符
- 需要头文件#include<string.h>
- 参数依次是指定字符串和待查字符
- 查找成功则返回指向找到字符的指针
- strchr()从左往右找，strrchr()从右往左找
- 字符串结束标记被认为是字符串的一部分
## 3.计算长度：strlen
- **功能**：求一个指定字符串的长度
- 需要头文件#include<stdio.h>
- 需要的参数为指定字符串
- 返回值不包括字符串结束标志
## 4.字符串切割：strtok()
- **功能**：将某个字符串按照指定的分隔符拆解为子串
- 需要头文件#include<stdio.h>
- 需要的参数依次为指定的字符串和指定的分隔符
- 执行成功返回指向子串的指针，否则返回空
- 该函数会改变原始字符串，使其所包含的所有分隔符变成结束标记
- 字符串指向的内存必须是可写的
## 5.字串拼接：strcat()与strncat()
- **功能**：将两个字符串拼接起来
- 需要头文件#include<stdio.h>
- 需要的参数依次为指向目标内存的指针，最终存储拼接之后的字符串；指向需要拼接的字符串的指针；规定最多拼接几个字节
- 返回拼接后的字符串
- strcat()没有边界控制，因此可能发生内存溢出
- strncat()有边界控制
## 6.字符串拷贝：strcpy()与strncpy()
- **功能**：复制字符串
- 需要头文件#include<stdio.h>
- 参数依次有指向目标内存的指针，指向待复制的字符串的指针，指定边界
- 返回一个指向复制后字符串的指针
## 7.字符串比较：strcmp()与strncmp()
- **功能**：比较两个字符串
- 需要头文件#include<stdio.h>
- 大于等于或小于分别返回1、0、-1
- 比较字符串大小实际上比较的是字符的ASCII码值的大小
- 从左到右逐个比较字符串的每一个字符，当能比较出大小时停止
