# 1. c进程与内存布局
## 虚拟内存
- 每个C语言进程都拥有一片结构相同的虚拟内存
- 虚拟内存是从实际物理内存映射出来的地址规范范围
- 所有的虚拟内存布局都是相同的，极大方便内核管理不同的进程
- **虚拟内存结构**：
	- 栈（stack）
		- **特点**：
			1. 运行时栈，运行过程中动态变化
			2. 大小有限
			3. 后进先出
		- **内容**：
			1. 环境变量
			2. 命令行参数
			3. 局部变量
		- 栈内存的分配与释放都是由系统规定的，程序员无法干预
	- 堆（heap）
		- **概念**：被称为动态内存、自由内存。开发者可以根据需要申请内存的大小、决定使用时间的长短等
		- 所有细节由开发者控制，系统不作任何干预
		- **基本特征**：
			- 堆的总大小仅受限于物理内存
			- 堆内存从下往上增长
			- 堆内存是匿名的，只能由指针来访问
			- 自定义分配的内存除非开发者主动释放，否则永不释放，直到程序退出
		- **相关API**：
			- 申请堆内存malloc()/calloc()：
				- 函数原型 ： void \*malloc(size_t size)  void \*calloc(size_t count,size_t size )
				- 参数：
					1. size：所申请的一块堆内存的大小，单位是字节
					2. count:所申请的堆内存块数，所有的内存块是连续分布的、无间隔的
				- 返回值：返回指向分配好的堆内存的指针
			- 清零指定内存bzero()：
				- 函数原型：void bzero(void \*s,size_t n);
				- 参数：
					1. s：指向要清零的内存的指针
					2. n要清零的内存大小，单位是字节
			- 释放堆内存free():
				- 函数原型： void free(void \*ptr);
				- 参数：ptr堆内存指针
			- 注意：
				- malloc()申请的堆内存，默认情况下是随机值，一般需要用bzero来清零
				- calloc()申请的堆内存，默认情况下是已经清了零的，不需要再清零
				- free()只能释放堆内存，并且只能释放整块堆内存，不能释放别的区域或者部分释放
			- 释放内存的含义：
				- 释放内存意味着将内存的使用权还给系统
				- 释放内存并不会改变指针的指向
				- 释放内存并不会对内存做任何修改，更不会将内存清零
	- 数据段
	- 代码段
## 静态数据
- **分类**：
	- 全局变量：定义在函数外部的变量
	- 静态局部变量：定义在函数内部且被static修饰
- **作用**：
	- 全局变量默认对所有文件可见，为某些需要在各个不同文件和函数间访问的数据提供操作上的便利
	- 静态局部变量可以在一个函数推出后依然保留局部变量的值
- **注意**：
	- 若定义时未初始化，则系统会将所有的静态数据自动初始化为0
	- 静态数据的初始化语句只会执行一遍
	- 静态数据从程序开始运行时就已存在，直到程序退出时才释放
