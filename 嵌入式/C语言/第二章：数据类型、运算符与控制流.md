# 1.整型
- **概念**：表达整数类型的数据
- **整型修饰符**：
    - short：短整型，用来缩短整型变量的尺寸
    - long：长整型，用来增长整型变量的尺寸
    - long long：长长整型，用来增长整型变量的尺寸
    - unsigned：用来去除整型变量的符号位，使得整型变量只能表达非负整数
- **格式控制符**：
	1. int整型：%d
	2. short整型：%hd
	3. long整型：%ld
	4. long long整型：%lld
	5. 显示不同进制的前缀：%#o（八进制）、%#x（十六进制）
- 使用整型修饰符后关键字int可以省略
- **符号位**：
	- 有符号的整型数据，首位为符号位，0表示正数，1表示负数
	- 无符号的整型数据，没有符号位
- **编码形式**：
	- 原码：正数直接使用二进制来表达
	- 补码：负数用绝对值取反加一来表达
	- 负数的补码在取反加一的时候，符号位是不动的
- **溢出**：
	- 超过数据所能表达的范围，称为溢出，且最大值和最小值是相邻的
- **进制**：源码中可以使用八进制、十进制或十六进制，但实际数据在内存中一律是二进制
# 2.浮点型、字符串和布尔型
## 2.1浮点型 
- **概念**：用来表达实数的数据类型
- **分类**：
	- 单精度浮点数（float），典型尺寸是四字节（%f）
	- 双精度浮点数（double），典型尺寸是八字节(%lf)
	- 长双精度浮点型（long double），典型尺寸是16字节(%Lf)
## 2.2字符串
### 字符
- char类型数据本质是单字节整型，支持整型的所有运算
- 计算机中存储的都是二进制的0和1，因此各种字符都必须被映射为某个数字才能存储到计算机中，这种映射关系形成的表就是ASCII码表
### 字符串
- **C语言两种表达字符串的方式**：
	1. 使用字符指针表示字符串
	2. 使用字符数组表示字符串
- **在内存中的存储**：
	- 在内存中实际上是多个连续字符的组合
	- 任何字符串都以一个“\0”作为结束标记
## 2.3布尔型数据
- **概念**：布尔型数据只有真假两种取值，非零为真，零为假
- **注意**：
	1. 除了1之外任何非零的自然数都可以表示真，等价于1
	2. 使用布尔型需要包含头文件stdbool.h
- 布尔型数据常用于逻辑判断、循环控制等场合
# 3.标准输入
- **概念**：从键盘中输入的数据被称为标准输入
- **相关函数**:
	- scanf()格式化输入函数（变量前需要加取址符）
	- fgets()字符串输入函数
- **注意**
	- scanf()与printf()不同，scanf()的格式控制串不能乱写，尤其是结尾处的\n
	- 用户必须完全按照scanf()中描述的输入格式来输入数据，否则将会出错
	- scanf()的返回值，代表从键盘读取的数据的个数
	- 无法匹配scanf()指定格式的数据，将被遗留在输入缓存区中，不会消失，会干扰后续程序的执行
# 4.类型转换与可移植性数据
## 类型转换
- **概念**：不一致但相互兼容的数据类型，在同一表达式中将发生类型转换
- **转换模式**：
	- 隐式转换：系统按照隐式的规则自动进行的转换
	- 强制转换：用户自定义进行的转换（可能丢失精度）
	- 隐式规则从小类型向大类型转换，目的是保证不丢失表达式中数据的精度
- 不论隐式转换还是强制类型转换，变换的都是在操作过程中的类型，是临时的，操作数本身的类型无法改变
- **数据类型的本质**：
	- 各种不同类型的数据本质上都是用户与系统对某一块内存数据的解释方式的约定
	- 推论：
		1. 类型转换，实际上是对先前做出的约定，做了一个临时的打破
		2. 理论上可以对任意的数据做任意的类型转换，但转换之后的数据解释不一定有意义
- **整型数据尺寸**
	- 概念：某种整型所占用的数据空间的大小
	- C语言中并未规定整形数据的具体大小，只规定了相互之间的相对大小
	- 系统字长：CPU一次处理的数据长度，称为字长
	- 同样的代码在不同的系统中可能由于数据尺寸发生变化而无法正常运行，因此系统标准整型数据类型是不可移植的
## 可移植性整型
- 概念：不管放到什么系统，尺寸保持不变的整型数据，称为可移植性整型
- 关键：typedef
- 思路：
	1. 为所有的系统提供一组固定的，能反映数据尺寸的、统一的可移植性整型名称
	2. 在不同的系统中，为这些可移植性整型提供对应的typedef语句
# 5.运算符
## 算术运算符
- **关注点**：
	- 取模运算要求左右两边的的操作数必须是整型数据
	- 自加和自减运算不仅可以对整型操作，也可以对浮点数、指针操作
- 前缀自加自减运算：先进行自加或自减，再参与表达式运算
- 后缀自加自减运算：先参与表达式的运算，再进行自加或自加自减
## 关系运算符
- **关注点**：
	1. 关系运算符用于判断运算符两边的表达式是否满足给定的大小条件
	2. 由关系运算符组成的表达式，其值为布尔型
## 逻辑运算符
- **运算规则**：
	1. 逻辑反（!）：将逻辑真假反转
	2. 逻辑与（&&）:将两个关系表达式串联起来，当且仅当左右两个表达式都为真时，结果为真
	3. 逻辑或（||）:将两个关系表达式并联起来，当且仅当左右两个表达式的结果都为假时，结果为假
## 位运算符
- 位运算符操作的对象是数据中的每一位
- **运算规则**：
	- 位逻辑反、位逻辑与、位逻辑或拥有与逻辑运算相似的规则和一样的真值表
	- 异或运算（^）：相同为0不同为1
	- 位移运算：移出去的不要，空出来的补零
- C语言没有规定有符号数的右移运算该如何操作
