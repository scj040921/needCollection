# 1. 作用域
## 基本概念
- C语言中，标识符都有一定的可见范围，这些可见范围保证了标识符只能在一个有限的区域内使用，这个可见范围就是作用域
- 尽量缩小表示符的作用域是一项软件开发中的基本原则
- 一个标识符的作用域超过它实际需要的范围时，会对整个软件的命名空间造成污染，导致一些不必要的冲突和误解。
## 函数声明作用域
- **概念**：在函数的声明式中定义的变量，其可见范围仅限于该声明式
- 参数名一般可省略，但一般不这么做
## 局部作用域
- **概念**：在代码块中定义的变量，其可见范围从其定义处开始到代码块结束为止
- **要点**：
	- 代码块是指一堆花括号括起来的区域
	- 代码块可以嵌套包含，外层的标识符会被内部的同名标识符临时掩盖而变得不可见
	- 代码块作用域的变量，由于其可见范围式局部的，所以被称为局部变量
## 全局作用域
- **概念**：在代码块外定义的变量，其可见范围可以跨越多个文件
# 2.作用域的临时掩盖
- 如果有多个不同的作用域互相嵌套，那么小范围的作用域范围会临时“遮蔽”大范围的作用域中的同名标识符，临时失去可见性。
- 尽量还是 要避免冲突
# 3. 存储期
## 基本概念
- 从内存分配到释放的时间间隔，一个变量的诞生到死亡就是变量的生命周期
- 生命周期的三种形式：
	- 自动存储期
	- 静态存储期
	- 自定义存储期
## 自动存储期
- 在栈内存中分配的变量，全部拥有自动存储期，因此也都被称为自动变量
- 内存管理过程是自动的
## 静态存储期
- 在数据段中分配的变量，统统拥有静态存储期，因此也都被称为静态变量。
- 这里静态的含义，指的是这些变量的不会因为程序的运行而发生临时性的分配和释放，它们的生命周期是恒定的，跟整个程序一致。
