# 1.数组
## 基本概念
1. **数组定义**
	- 逻辑：一次性定义多个相同类型的变量并存储到一片连续的内存中
2. **元素引用**
	- 数组元素的引用，指的是如何使用数组中的某个元素，数组中的元素都是有连续编号的，被称为数组下标，从零开始依次递增
	- 存储模式：一片连续的内存，按数据类型分割成若干大小相同的格子
3. **数组赋值**：
	- 数组的本质是一系列连续存储的变量，是可以对其进行赋值的，使用上述元素引用的语法即可。但是数组作为一个整体是不可以直接赋值的
	- 赋值方式：
		1. 使用数组初始化，直接提供一系列初始值进行赋值
		2. 使用循环逐个元素赋值
		3. 使用memcpy函数，将一整块内存整体复制到另一块内存
## 特殊数组
1. **字符数组**：专门存放字符的数组
2. **多维数组**：若数组元素类型也是数组，则该数组称为多维数组，语法与一维数组完全一致
## 数组语法解剖
- 任意的数组不管有多复杂，其定义都有两部分组成
	- 数组主体：说明数组名和元素个数
	- 元素类型：说明元素的类型，可以是任意的类型
- 元素类型如果由多个单词组成，C语言规定需要将其拆散写到数组主体两边
## 数组转换为指针
**除非有sizeof或取址符，否则数组将一律被转换成一个指向其首元素的指针**
## 数组下标运算符
- 数组下标实际上是编译系统的一种简写，其等价形式是：

```
a[i] = 100;  等价于  *(a+i) = 100;
```
- 根据加法交换律，以下的所有的语句均是等价的：

```
  a[i] = 100;
*(a+i) = 100;
*(i+a) = 100;
  i[a] = 100;
```
- 数组运算等价于指针运算
## 字符串常量
- 字符串常量在内存中的存储，实质上是一个匿名数组
- 匿名数组，同样满足两种含义的规定
## 特殊数组
1. **零长数组（柔性数组）**
	- 概念：长度为0的数组
	- 用途：放在结构体的末尾，作为可变长度数据的入口
2. **变长数组**
	- 概念：定义时使用变量作为元素个数的数组
	- 变长数组仅仅指元素个数在定义时时变量，而不是只数组的长度可长可短。不论是普通数组还是变长数组，数组一旦定义完毕，其长度则不可改变
# 2.指针
## 准备知识
1. **内存地址**：
	- 字节：字节是内存的容量单位，一个字节由八位
	- 地址：系统为了区分每一个字节而对他们逐一进行的编号，称为内存地址，简称地址
2. **基地址**：
	- 单字节数据：对于单字节数据而言，基地址就是其字节编号
	- 多字节数据：对于多字节数据而言，基地址编号是其所有字节中编号最小的那个
3. **取址符**：
	- 每个变量都是一块内存，都可以通过取址符&获取其地址
	- 虽然不同的变量的尺寸不同，但是他们地址的尺寸是一样的
	- 不同的内存地址虽然形式上看起来是一样的，但由于他们代表的内存尺寸和类型都不同，因此他们在逻辑上是严格区分的
## 指针入门
1. **指针的概念**：
	- 指针就是地址：
	- 指针就是指针变量
2. **指针的赋值**赋给指针的地址，类型需要跟指针的类型相匹配
3. **指针目标的索引**：
   所谓索引，指的是通过指针，取得其指向的目标
	- 指针尺寸指的是指针所占内存的字节数
	- 指针所占内存取决于地址的长度，而地址的长度取决于系统寻址范围，即字长，所以指针尺寸只与系统的字长有关，跟具体的指针类型无关
## 特殊指针
1. **野指针**：
	- 概念：指向一块未知区域的指针被称为野指针，野指针是危险的。
	- 危害：
		1. 引用野指针相当于访问了非法的内存，常常会导致段错误
		2. 引用野指针可能会破坏系统的关键数据，导致系统崩溃等严重后果
	- 产生原因：
		1. 指针定义之后未初始化
		2. 指针所指的内存被系统回收
		3. 指针越界
	- 如何防止：
		1. 指针定义时及时初始化
		2. 绝不引用已被系统回收的内存
		3. 确认所申请的内存边界，谨防越界
2. **空指针**：
   空指针指的是指向零地址处的指针，并非时指针里面什么都没有，而是说指针变量内存储了地址0
	- 特点：零地址处的内存不可访问，不应对应任何系统数据，指针指向这块内存就算误操作也不会对系统造成任何危害。因此空指针最大的特点是安全。
	- 不可避免地遇到野指针时，一般地做法就是将这些危险地野指针指向零地址，使之成为空指针
## 指针语法解剖
- 任意的指针不管有多复杂，其定义都由两部分组成
	- 指针主体部分：一个星号，和一个变量的名称，星号意味着该变量专用于存储地址，即指针变量
	- 模板类型部分：指针所指向的数据类型，可以是任意的类型
- 目标类型如果由多个单词组成，C语言规定需要将其拆散写到指针变量主体的两边
## 指针运算
- 指针加法意味着地址向上移动若干个目标
- 指针减法意味着地址向下移动若干个目标
## 特殊指针
1. **char型指针**
   char型指针实质上与其它类型的指针并无本质区别，但由于C语言中的字符串以字符数组的方式存储，而数组在大多数场合又会表现为指针，因此字符串在绝大多数场合就表现为char型指针
2. **多级指针**
	- 如果一个指针变量 p1 存储的地址，是另一个普通变量 a 的地址，那么称 p1 为一级指针
	- 如果一个指针变量 p2 存储的地址，是指针变量 p1 的地址，那么称 p2 为二级指针
	- 如果一个指针变量 p3 存储的地址，是指针变量 p2 的地址，那么称 p3 为三级指针
	- 以此类推，p2、p3等指针被称为多级指针
3. **void指针**
	- 概念：无法明确指针所指向的数据类型时，可以将指针定义为void型指针
	- 要点：
		1. void型指针无法直接索引目标，必须将其转换为一种具体类型的指针方可索引目标
		2. void型指针无法进行加减运算
	- 作用：
		1. 修饰指针表示指针指向一个类型未知的数据
		2. 修饰函数参数列表，表示函数不接受任何参数
		3. 修饰函数返回类型，表示函数不返回任何数据
4. **const指针**
	- const型指针有两种形式：1. 常指针 2. 常目标指针
		1. 常指针const修饰指针本身，表示指针变量本身无法修改
		2. 常目标指针const修饰指针的目标，表示无法通过该指针修改其目标
	- 常指针在实际应用中并不常见
	- 常目标指针早实际应用中广泛可见，用来限制指针的读写权限
5. **函数指针**
	- 概念：指向函数的指针
	- 函数指针和普通指针在本质上并无区别，只是在取址和索引时，取址符和星号均可省略
	- 要点：
		1. 函数指针是一类专门用来指向某种类型函数的指针
		2. 函数的类型不同，所需要的函数指针也不同
		3. 函数的类型，与普通变量的类型判定一致，即去除声明语句中的标识符之后所剩的语句
