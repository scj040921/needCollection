# 1.结构体
## 基本概念 
- 结构体是c语言中用来将多种不同的数据类型组合起来，形成某种现实意义的自定义的变量类型，本质上是一种自定义类型
	```
	struct 结构体标签
	{
		成员1;
		成员2;
		...
	};
	```
- **语法**：
	- 结构体标签用来区分各个不同的结构体
	- 成员是包含在结构体内部的数据，可以是任意的数据类型
- **注意**：
	- 空结构体的大小不为0
	- 因为内存对齐机制，sizeof的结果不等于成员大小之和
## 结构体初始化
- **结构体的定义和初始化**：
	- 结构体内部拥有多个不同类型的成员，初始化采用与数组类似的列表方式
	- 初始化方式：
		1. 普通初始化
		2. 指定成员初始化
- **指定成员初始化的好处**：
	- 成员初始化的次序可以改变
	- 可以初始化一部分成员
	- 结构体新增成员之后初始化语句依然可用
## 结构体成员引用
结构体相当于一个集合，其中的每个成员都是独立的变量，都可以被独立的引用
## 结构体数组与结构体指针
- 结构体指针：
	- - 当变量是**指针**时，必须用 `->`；当变量是**实体**时，必须用 `.`
	- **声明不等于分配**：`struct Node *p;` 仅仅声明了一个指针，**没有**为结构体分配内存。此时对 `p->data` 赋值是**未定义行为**
	- **指针加法按"结构体大小"步进**：`ptr + 1` 移动的是 `sizeof(struct X)` 字节，而不是1字节。
	- - **浅拷贝导致的共享问题**：两个指针指向同一结构体，修改一个会影响另一个。若需独立副本，必须深拷贝。
```
struct node  n = {100, 'x', 3.14};
struct node *p = &n;

// 以下语句都是等价的
printf("%d\n",   n.a);
printf("%d\n", (*p).a);
printf("%d\n",  p->a);  // 箭头 -> 是结构体指针的成员引用符
```
- 结构体数组：
```
struct node s[5];
s[0].a = 300;
s[0].b = 'z';
s[0].c = 3.45;
```
#   2.结构体位域
## 位域的定义
- 位域允许我们自定义结构体成员的长度，只需要在成员后加上冒号后面跟上结构体成员的长度，单位是bit。
- **注意事项**：
	- 位域成员必须是整型类型
	- 位域成员的长度不能超过其数据类型的长度。
	- 位域成员声明中的类型，规定了对应位域最大的比特位位数
	- 位域各成员，在内存中是按照声明的顺序连续存储的
	- 位域成员不可取址（原因：位域成员可能不占用完整的字节，甚至不从一个字节边界开始，而可寻址的单元的最小粒度就是一个字节，**位域是编译器提供的语法糖**：当我们写 `sh.fan = 0;` 时，编译器会自动把它转换成“读出包含 `fan` 的字节或字，通过位操作修改对应比特，再写回去”。这个过程对程序员透明，但位域本身只是一个名字，不是一个独立的存储对象。）
## 位域的使用
- 对结构体中的位域变量的操作与普通结构体成员没什么区别
- 使用位域的好处一方面是节省内存的占用，里另一方面在读取时不需要移位运算就可以直接读取某个字节内部的某些比特位数值
# 3.结构体尺寸
## 地址对齐
- 数据在内存中的存储地址最好可以与系统的操作单位长度“对齐”（一个类型大小为 N 字节的数据，其**起始地址应该是 N 的整数倍**这叫做“自然对齐”或者起始地址是访问粒度的整数倍）
- 通过地址对齐可以减少读取时的操作次数，在尽可能少的读取次数内完成读取，如果发生数据地址未对齐的情况，有些系统会直接罢工，有些系统则降低性能
## M值
-  变量的m值是变量的地址和尺寸的约束条件
1. **普通变量的m值**：
	- 对于一个尺寸固定的数据而言，当它的地址满足某个数的整数倍时，就可以保证地址对齐。这个数就被称为变量的m值。
	- 一般情况下，数据本身尺寸就是它的m值
2. **结构体的M值**：
	- 结构体的M值取决于其成员的m值的最大值
	- 结构体的地址和尺寸，都必须等于M值的整数倍
3. **修改变量m的值**：
	```
	char c __attribute__((aligned(32))); // 将变量 c 的m值设置为32
	```
- m值反映一个变量的起始地址的要求，这个要求可以通过attribute语法来修改
- **要点**：
	- attribute机制是GNU特定语法，属于C语言标准语法的扩展
	- attribute前后都是双下划线，aligned两边是双圆括号
	- attribute语句出现在变量定义语句中的分号前面，变量标识符后面
	- attribute机制支持多种属性设置，其中aligned用来设置变量的m值属性
	- 一个变量的m值只能提升，不能降低，且只能为正的2的n次幂
- 注意：修改变量的m值只是修改了对变量起始地址的要求
## 可移植性
- 可移植指定是同样的代码或数据在不同的平台上都可以正常运行
- 对于数据来说，有两方面可能会导致不可移植：
	- 数据尺寸发生变化
	- 数据位置发生变化
